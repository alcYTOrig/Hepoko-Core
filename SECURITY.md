## Reporting a Vulnerability

Use this section to tell people how to report a vulnerability.

Tell them where to go, how often they can expect to get an update on a
reported vulnerability, what to expect if the vulnerability is accepted or
declined, etc.

### How to Report

If you discover a security vulnerability in HepokoCore, please report it via one of the following methods:

- **Email**: Send a detailed report to `security@alcytorig.com` (if available) or to the maintainer's email
- **Discord**: Contact the project maintainer via Discord (if available)
- **GitHub Issues**: Create a private security advisory in this repository

### What to Include

Please include the following information in your report:

- **Version** of HepokoCore affected
- **Minecraft version** and **server software** (Paper, Purpur, etc.)
- **Steps to reproduce** the vulnerability
- **Impact** of the vulnerability (e.g., privilege escalation, data leak, etc.)
- **Proof of concept** (if available)
- **Suggested fix** (if you have one)

### Response Time

You can expect an initial response within **48-72 hours** after submitting a vulnerability report. We will do our best to address critical vulnerabilities as quickly as possible.

### Security Update Process

1. Vulnerability is confirmed and triaged
2. A fix is developed and tested
3. A new version is released with the fix
4. Users are notified via GitHub releases and Discord (if applicable)

### Supported Versions

Only the **latest major version** and **one previous major version** receive security updates. We strongly recommend updating to the latest version to ensure you have all security fixes.

## Security Best Practices

When using HepokoCore:

- Always keep your server software (Paper/Purpur) up to date
- Regularly update all plugins, including HepokoCore
- Use strong passwords for authentication
- Configure permissions carefully
- Monitor server logs for suspicious activity
- Use HTTPS for any web-based integrations
